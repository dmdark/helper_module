=== РАБОТА С ФОРМАМИ ===

Формы настраиваются в "config.php" в секции $config['adminConfig']['forms'] примерно следующим массивом:
----------
'forms' => array(
   // Форма 1
   array(
      'Параметры формы' => 'значение'
      // Массив полей формы
      'fields' => array( // Список полей
         // Поле 1
         array(
            'Параметры поля' => 'значение'
         ),
         // Поле 2 и т.д.
         array(
            'Параметры поля' => 'значение'
         ),
      ),
   ),
),
----------



=== Параметры формы ===

! - обязательный параметр (в config.php прописывается без "!")

!tag:           тег на странице (<!--tag-->), заменяемый на форме; также используется как имя каталога для сохраняемых писем, как id формы и ее элементов
header:         заголовок формы, отображается в теге <h2 />
submit:         надпись на кнопке отправки, по умолчанию "Отправить"
success:        сообщение, отображаемое при успешной отправке письма; если не указать, форма просто исчезнет
errors:         общее сообщение об ошибке валидации формы
css:            имя CSS файла (с расширением) или false, если не требуется подключать; файл берется из "/_seo/modules/forms/styles/filename", подключается не сам файл, а его содержимое в тег <style />
!emailSettings: массив с параметрами отправки письма
!from:          отправитель письма, формат "mail@example.com" или "Имя <mail@example.com>"
!emails:        получатели письма, обязательно в формате "mail@example.com", через запятую, без пробелов
letter_subject: тема письма
letter_intro:   вводный текст письма
!fields:        массив со списком полей



=== Параметры письма ===

! - обязательный параметр (в config.php прописывается без "!")

!from:   отправитель письма, формат "mail@example.com" или "Имя <mail@example.com>"
!to:     получатели письма, обязательно в формате "mail@example.com", через запятую, без пробелов
subject: тема письма
intro:   вводный текст письма



=== Параметры поля ===

! - обязательный параметр (в config.php прописывается без "!")

!name:       значение атрибута "name", эти значения для любых двух полей не должны совпадать
letter_name: имя поля в письме
tag:         input/textarea - тег поля, по умолчанию input
label:       текст тега <label />
placeholder: значение атрибута "placeholder"
type:        тип для поля input, по умолчанию text. Не поддерживается checkbox и radio!
validator:   список проверок в формате 'notempty,max=100'; допустимые варианты: 'min/max=int', 'notempty' (не пусто), 'email', 'phone'; лучше в любом случае указывать 'max=X'
required:    true/false, влияет только на вывод "*" в <label />
error:       сообщение при ошибке валидации данного поля
